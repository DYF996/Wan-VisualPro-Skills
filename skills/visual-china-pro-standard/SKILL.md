## ⚙️ 4维质量评估标准

每次输出前，必须自检以下4个维度：

### 维度1：商业传达力（权重30%）
- [ ] 主体明确，第一眼能看懂要表达什么
- [ ] 信息层级清晰，视觉引导逻辑正确
- [ ] 品牌/产品识别度高

### 维度2：视觉冲击力（权重30%）
- [ ] 光影具有立体感和戏剧性
- [ ] 色彩搭配具有视觉吸引力
- [ ] 整体风格与竞品有明显差异

### 维度3：技术完成度（权重25%）
- [ ] 分辨率达标，无锯齿/模糊
- [ ] 色彩准确，无色偏/溢出
- [ ] 细节丰富，无塑料感/涂抹感

### 维度4：商业合规性（权重15%）
- [ ] 无版权争议元素
- [ ] 符合广告法要求
- [ ] 适配目标平台的格式规范

## 📋 标准工作流程（4步法）

### 步骤1：场景识别与参数加载

**📝 操作：**
1. 识别用户上传图片的主要元素
2. 推荐最匹配的S1-S6场景
3. 自动加载对应场景的4大模块预设参数
4. 询问用户是否需要调整（默认使用预设）

### 步骤2：智能分析

**📝 操作：**
```
输出内容：
┌─────────────────────────────────────────┐
│  📊 图片分析报告                          │
├─────────────────────────────────────────┤
│  主体识别：[AI识别的主体元素]              │
│  当前光影：✅自然光 / ⚠️平乏 / ❌逆光      │
│  当前色彩：✅正常 / ⚠️偏色 / ❌杂乱        │
│  当前构图：✅合理 / ⚠️偏移 / ❌杂乱        │
│  场景匹配：推荐 [场景编号] - [场景名称]    │
└─────────────────────────────────────────┘
```

### 步骤3：自动化渲染

**📝 操作：**
按照选定场景的4大模块参数，自动生成优化后的Prompt：
[主体识别] + [场景特征] + [光影参数] + [色彩色卡] + [质感标准] + [构图规则]

### 步骤4：质量验收

**📝 操作：**
```
输出4维评分：
- 商业传达力：★×N
- 视觉冲击力：★×N  
- 技术完成度：★×N
- 商业合规性：★×N

综合评级：[A+/A/B/C]
可商用建议：[是/否 + 原因]
```

## 🔌 API调用协议

### Wan2.7-Image 模型调用

**端点：** `/api/image/generate`

**请求格式：**
```json
{
  "model": "wanx2.7-image",
  "prompt": "[完整Prompt]",
  "negative_prompt": "low quality, blurry, watermark, text, logo, trademark",
  "aspect_ratio": "1:1/3:4/4:3/16:9/9:16",
  "resolution": 1024,
  "style": "commercial_photography",
  "quality": "hd"
}
```

**标准negative_prompt：**
```
blurry, low resolution, pixelated, watermark, text, logo, trademark,
distorted face, deformed hands, extra limbs, bad anatomy,
ugly, oversaturated, washed out colors, plastic skin,
amateur, unprofessional, stock photo look
```
